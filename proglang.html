Programming Languages

Term Definition
Machine Language Instructions encoded as bit patterns along with the encoding system
Debugging Locating bugs, errors in the code, and correcting them
Assembler Programs that were developed to convert mnemonic expressions into machine language instructions
Assembler Language A mnemonic system for representing programs. Became known as second-generation languages
Translator A program written that translated programs expressed in high-level primitives into machine-language programs
Compiler A translater program that often had to compile several machine instructions into short sequences to simulate the activity requested by a single high-level primitive
Interpreter Programs that executed the instructions as they were translated instead of recording the translated version for future use. Executed a program from its high-level form
Cross-platform Software Software that is independent of an operating system’s design as well as the machine’s hardware design and is therefore executable throughout a network.
Natural Language Language that involves more than responding to a few rigorously defined primitives. Evolved over time without formal grammicatical anaylsis. (like English, German, and Latin)
Formal Language Language precisely defined by grammar (such as programming languages)
Machine Independence Third Gen language that did not rely on the characteristics of a particular machine
High-level Programming Language Language code can leverage millions of lines of library code, invoke built-in operating system functions, or initiate complex interactions with networks and databases. Languages that make computer programs both easier for human authors to write, and easier for subsequent readers to understand
Object-oriented Programming (OOP) A software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects
Scripting Language These languages are typically used to perform administrative tasks rather than to develop complex programs
Interger Data Type Numeric data consisting of whole numbers, probably stored using two’s complement notation. Operations that can be performed on integer data include the traditional arithmetic operations and comparisons of relative size, such as determining whether one value is greater than another
Float Data Type Numeric data that might contain values other than whole numbers, probably stored in floating-point notation. Operations performed on data of type float are similar to those performed on data of type integer but differ because float requires 2 integers
String Data Type Character refers to data consisting of symbols, probably stored using ASCII or Unicode. Operations performed on such data include comparisons such as determining whether one symbol occurs before another in alphabetical order, testing to see whether one string of symbols appears inside another, and concatenating one string of symbols at the end of another to form one long string.
Primitive Data Type The types integer, float, character, and Boolean are common primitives. Other data types that have not yet become widespread primitives include images, audio, video, and hypertext
Data Structure The conceptual shape or arrangement of data
Python List Data Structure Items that are ordered, changeable, and allow duplicate values. They are index with first one having number 0 and second item having number 1
Python Tuple Data Structure A collection which is ordered and unchangeable
Python Set Data Structure A collection which is unordered, unchangeable*, and unindexed. Does not allow duplicate values
Python Dictionary Data Structure Item that are ordered, changeable, and does not allow duplicates. presented in key:value pairs, and can be referred to by using the key name
Function A set of instructions for performing a task that can be used as an abstract tool by other program units
Local Variable A variable declared within a function
Global Variable Variables whose scopes are not restricted to a particular part of a program
Source Program A program in its original form
Object Program A program in its translated form
