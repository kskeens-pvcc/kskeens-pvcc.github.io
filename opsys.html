Operating Systems Terminology

Term Definition
Operating System The software that controls the overall operations of a computer. It executes user programs, coordinate activites, and manage system resources.
Job Queue A number of programs(jobs) in mass storage waiting to be executed by the operating system
FIFO First-in, First-out. Objects(jobs) removed from the queue in the order they arrived.
Interactive Processing This allowed a program being executed to carry on a dialogue with the user through remote terminals.
Time-sharing An operating system that provided service to multiple users at the same time.
Multiprogramming A technique that divided time into intervals and then executed each job resticting it to only one interval at a time.
Application Software It consists of the programs for performing tasks particular to the machine's utilization.
System Software It provides that infrasture that the application software requires.
Utility Software It consists of programs for performing activities that are fundemental to computer installations but not included in the operating system. ALso consists of software unites that extend the capabilites of the operating system.
Graphic User Interface (GUI) Objects to be manipulated, such as files and programs, are represented pictorially on the display as icons. These systems allow users to issue commands by using on of several input devices.
Shell A communication with users through textual messages using keyboard and monitor screens.
Kernel This contains the software components that perform the very basic functions required by the computer installation.
Device Driver The software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
File Manager This unit's job is to coordinate the ise of the machine's mass storage facilities. It maintains records of all the filies stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extenions to existing files.
Memory Manager Component charged with the task of coordinating the machines use of main memory.
Virtual Memory The large "fictional" memory space created by paging.
Boot Strapping The procedure that tranfers the operating system form mass storage into main memory.
ROM Read-only Memory. A portion of the computer's memory where the CPU expects to find its intial program is contructed from special nonvolitile memory cells. The contents can be read but not altered.
Deadlock The condition in which two or more processes are blocked from progressing becuase each is waiting for a resource that is allocated to another.
Spooling The technique of holding data for output at a later but more convient time.
Super User (or Administrator) A person that gains high privelaged access to the operating system by identifying him- or herself as the administaor during the login procedure. The administraor can alter settings within the operating system, modify critical software packages, adjust the privelages granted to other users, and perform a varity of other maintenance activities that are denied normal users.
Multi-factor Authentification A type of authentification that requires more than one piece of evidence to gain access tosecure accounts.
